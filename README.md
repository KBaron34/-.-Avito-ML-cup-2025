# Поиск дублей|Avito ML cup 2025
Задача — разработать алгоритм, который на основе текстовой информации, изображений и других атрибутов позволит выявлять дубли в массиве объявлений.

Ссылка на соревнования: https://ods.ai/competitions/avitotechmlchallenge2025_2

## Описание задачи
В этом соревновании предстояло разработать алгоритм для автоматической детекции дубликатов объявлений  по их названию и фотографиям.
Продавцы могут описывать один и тот же товар разными словами, а фотографии могут различаться по ракурсу, фону или качеству. Например, допустимо, когда продавец размещает отдельные объявления для разных размеров одной модели кроссовок с практически идентичными описаниями и фотографиями. Однако повторная публикация одного и того же товара, даже с измененным текстом или другими фотографиями, считается дублем.

## Данные
Для решения задачи “Labelcraft” предлагается несколько групп данных и материалов:
1. Тренировочные данные
   - train_part_[0001-0004].snappy.parquet — выборка для обучения, содержащая 1.87 миллионов пар объявлений (4 файла по ~500 тыс. объявлений)
   - train_images_part_[0001-0004]-chunk[0001-0008].zip — 2.5 миллиона изображений в формате JPEG для соответствующих объявлений (30 файлов по ~100 тыс. изображений)

2. Тестовые данные
   - test_part_[0001-0002].snappy.parquet — тестовая выборка (2 файла по 250 тыс. объявлений).
   - test_images_part_[0001-0002]-chunk[0001-0007].zip — изображения в формате JPEG для соответствующих объявленийtest_images_part_[0001-0002]-chunk[0001-0007].zip — изображения в формате JPEG для соответствующих объявлений


### Описание колонок для тренировочной и тестовой выборок: 

   - base_item_id	id опорного объявления
   - cand_item_id	id объявления-кандидата
   - group_id	Номер группы пользователей, к которой принадлежит пара объявлений
   - action_date	Дата
   - base_category_name	Название категории опорного объявления
   - cand_category_name	Название категории объявления-кандидата
   - base_subcategory_name	Название подкатегории опорного объявления
   - cand_subcategory_name	Название подкатегории объявления-кандидата
   - base_param1	Первый уровень детализации подкатегории для опорного объявления
   - cand_param1	Первый уровень детализации подкатегории для объявления-кандидата
   - base_param2	Второй уровень детализации подкатегории для опорного объявления
   - cand_param2	Второй уровень детализации подкатегории для объявления-кандидата
   - base_title	Заголовок опорного объявления
   - cand_title	Заголовок объявления-кандидата
   - base_description	Описание опорного объявления
   - cand_description	Описание объявления-кандидата
   - base_price	Цена, указанная в опорном объявлении
   - cand_price	Цена, указанная в объявлении-кандидате
   - base_json_params	Атрибуты опорного объявления
   - cand_json_params	Атрибуты объявления-кандидата
   - base_count_images	Число фотографий в опорном объявлении
   - cand_count_images	Число фотографий в объявлении-кандидате
   - base_title_image	id титульной фотографии для опорного объявления
   - cand_title_image	id титульной фотографии для объявления-кандидата
   - is_same_location	Флаг совпадения локации (город, село и пр.) для пары объявлений
   - is_same_region	Флаг совпадения региона для пары объявлений
   - is_double	Является ли пара объявлений дублем
